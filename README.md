# تقویم شمسی هوشمند (Persian Smart Calendar PWA)

## معرفی
یک وب‌اپلیکیشن PWA کامل، آفلاین‌محور و ریسپانسیو برای نمایش تقویم شمسی
(همراه با نمایش تقویم میلادی و قمری)، مناسبت‌های روزانه، تبدیل تاریخ بین
سه تقویم، محاسبه اوقات شرعی و محاسبه سن. تم روشن/تاریک به‌صورت خودکار و
دستی قابل تغییر است.

## معماری (Architecture)

| فایل | نقش | الگوی طراحی |
|---|---|---|
| `js/jalali.js` | تبدیل دقیق شمسی↔میلادی (الگوریتم مرجع jalaali-js) | Static Utility + Value Object |
| `js/hijri.js` | تبدیل قمری↔میلادی (الگوریتم جدولی کویتی) | Static Utility + Value Object |
| `js/events.js` | مناسبت‌های ثابت شمسی/میلادی (بدون نیاز به سرویس) | Repository Pattern |
| `js/events-service.js` | دریافت مناسبت‌های قمری/رسمی از API + کش ۳۰ روزه در localStorage | Decorator + Cache-Aside |
| `js/prayer-times.js` | محاسبه اوقات شرعی بر پایه فرمول موضع خورشید (بدون API) | Strategy Pattern |
| `js/age-calculator.js` | محاسبه سن دقیق شمسی | Service (SRP) |
| `js/theme.js` | مدیریت تم روشن/تاریک | Observer Pattern |
| `js/app.js` | هماهنگ‌کننده کل اپ (Controllers) | Controller / Facade |
| `sw.js` | سرویس‌ورکر برای PWA و کش آفلاین | Cache-First / Network-First Hybrid |

## چرا فقط بخشی از داده از سرویس خارجی گرفته می‌شود؟
- تبدیل شمسی/میلادی/قمری، محاسبه اوقات شرعی، محاسبه سن: **کاملاً محلی و
  بدون هیچ API**، چون الگوریتم‌های ریاضی و نجومی قطعی هستند.
- مناسبت‌های ثابت (نوروز، سیزده‌به‌در، روزهای جهانی معروف): **محلی و
  هاردکد شده** در `events.js`.
- تنها مناسبت‌های **قمری/رسمی متغیر** (که به دلیل ماهیت قمری هرساله حدود
  ۱۰-۱۱ روز شیفت می‌کنند) از API رایگان
  [pnldev.com](https://pnldev.com/fa/api-doc/calender) گرفته شده و نتیجه
  در `localStorage` به‌مدت **۳۰ روز کش** می‌شود تا در بازدیدهای بعدی از
  همان ماه، نیازی به تماس مجدد با سرویس نباشد.

## اجرای پروژه
هیچ نصب یا build لازم نیست:

```bash
npx serve project
# یا
python3 -m http.server 8080 --directory project
```

توجه: برای فعال شدن Service Worker و نصب PWA باید روی HTTPS یا
localhost اجرا شود (محدودیت امنیتی مرورگرها).

## محدودیت‌های شناخته‌شده
- اوقات شرعی بر پایه فرمول نجومی PrayTimes.org محاسبه می‌شود و ممکن است
  با اعلام رسمی مراجع دینی چند دقیقه تفاوت داشته باشد.
- مناسبت‌های قمری متغیر برای اولین بازدید هر ماه به اینترنت نیاز دارند
  (سپس تا ۳۰ روز کش می‌شوند).
- تقویم قمری از الگوریتم جدولی (تقریبی) استفاده می‌کند؛ رؤیت هلال محلی
  می‌تواند ۱ روز تفاوت ایجاد کند.
